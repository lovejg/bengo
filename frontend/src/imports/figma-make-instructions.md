✅ Figma Make 작업 지시 (맞춤정책 상단 오른쪽 영역 채우기)

목표:
‘맞춤정책 결과’ 페이지 상단 히어로 영역에서, 왼쪽에는 제목/설명 텍스트가 있고 오른쪽(현재 비어있는 큰 공간)에 요약 + CTA 카드를 배치하여 정보 밀도를 높이고 다음 액션(조건 수정/프로필 보완)을 유도한다.
헤더(nav) 영역이 아니라, 제목이 있는 상단 섹션 내부 오른쪽 공간을 채운다.

1) 배치 규칙

상단 섹션을 2컬럼으로 구성:

Left: 기존 제목/설명 그대로 유지

Right: 새 컴포넌트 배치

데스크탑: Right 컬럼 폭은 전체의 약 40~45%

모바일: Right 컴포넌트는 제목 아래로 내려가서 1컬럼(세로 스택)

2) 새 컴포넌트: Hero / MatchSummaryCard

형태: 카드 1장(라운드/그림자는 기존 카드 시스템 재사용)
높이: 왼쪽 텍스트 블록과 시각적으로 균형(너무 크지 않게)

카드 내부 구조

상단 라벨(작게): 내 조건 요약

조건 칩(Chip) 4~6개 (줄바꿈 허용)

예시 칩: 28세, 강남구, 청년, 구직중(선택), 소득 미입력, 가구원 미입력

“미입력”은 시각적으로 덜 강조(연한 톤)하여 입력 유도 느낌

추천 정확도/완성도 영역

텍스트: 프로필 완성도 60%

진행바(Progress) 1개

서브 문장: 정보를 더 입력하면 정확한 추천을 받을 수 있어요

CTA 버튼 2개 (가로 배치)

Primary: 조건 수정하기

Secondary: 프로필 보완하기

버튼 크기는 헤더 버튼보다 한 단계 작게(하지만 클릭 가능성 확보)

3) 상태 Variants (컴포넌트화)

Hero / MatchSummaryCard를 Component Set으로 만들고 아래 variant 제공:

state=complete : 미입력 칩 거의 없음 / 완성도 90%+

state=partial : 미입력 칩 1~3개 / 완성도 40~80% (기본)

state=empty : 대부분 미입력 / 완성도 0~30% (프로필 보완 CTA 강조)

4) 보조 시각 요소(선택)

카드 오른쪽 상단 또는 배경에 아주 은은한 장식(도형/패턴)을 넣어 “빈 공간” 느낌을 줄인다.
단, 새 색상 정의 금지. 기존 브랜드 블루/그라데이션 톤을 매우 약하게만 사용.

5) 산출물

상단 섹션 오른쪽 영역에 Hero / MatchSummaryCard 배치 완료

반응형(데스크탑 2컬럼, 모바일 1컬럼)까지 고려한 레이아웃 완료

칩/진행바/버튼은 재사용 가능한 서브 컴포넌트로 정리

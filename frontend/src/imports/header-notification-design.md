목표:
현재 헤더(상단 네비)의 오른쪽 영역이 휑하므로, 우상단에 들어갈 알림(🔔) 컴포넌트를 디자인 컴포넌트화해서 추가한다.
MY 페이지와 역할이 겹치지 않도록, 헤더에서는 “최근 알림 미리보기 + 빠른 액션 + 더보기 링크”까지만 제공한다.

1) 컴포넌트 구성 (Component Set)

이름: Header / NotificationBell

Variant(필수):

state=default (알림 없음, 뱃지 없음)

state=hasUnread (미확인 알림 있음, 뱃지 N 또는 점)

state=open (드롭다운 열린 상태)

state=openEmpty (드롭다운 열렸는데 알림 0개)

state=openLoading (로딩 스켈레톤)

2) 헤더 배치 규칙

위치: 헤더 우측 영역에서 “로그인/회원가입” 왼쪽에 배치

정렬: 세로 중앙 정렬

간격: 알림 아이콘과 다음 요소(로그인 버튼) 사이 간격 12~16px

클릭 영역: 아이콘 자체 크기보다 크게(최소 40x40)

3) 알림 아이콘 디자인

기본: 종(🔔) 아이콘 1개

hover/active 상태: 배경에 아주 약한 원형 하이라이트

미확인 상태:

우측 상단에 작은 뱃지(원형 or pill)

숫자는 1~99 표시, 99 초과는 “99+”

알림 개수 0이면 뱃지 표시하지 않음

4) 드롭다운(팝오버) UI 사양

컴포넌트 이름: NotificationPopover

크기/레이아웃

폭: 340px (±20 허용)

높이: 자동 (최대 520px, 넘치면 내부 스크롤)

모서리: 기존 카드들과 동일한 라운드

그림자: 기존 카드/모달과 동일한 elevation

상단 헤더 영역

좌측: “알림”

우측: 텍스트 버튼 “모두 읽음”

헤더 아래 얇은 구분선

리스트 영역

최대 6개까지만 보여주고, 초과는 스크롤

알림 아이템 간 세로 간격 8~10px

아이템 패딩 12~14px

알림 아이템(한 줄 카드 형태)

각 아이템은 아래 요소로 구성:

(좌측) 타입 아이콘 or 타입 라벨 chip (선택)

(본문 1줄) 제목: 예) [D-3] 청년 월세 지원 사업 마감 임박

(본문 2줄) 서브텍스트: 예) 청년 월세 지원 사업 · 2026.03.28

(우측/하단) 1개의 액션 버튼(작은 사이즈)

예: 바로 신청 / 필요서류 보기 / 조건 보완

미확인 알림은 왼쪽에 작은 점 또는 배경 강조(아주 약하게)

하단 푸터

링크 스타일: “알림 더보기 →”

클릭 시 MY의 알림센터/내 소식으로 이동하는 느낌의 링크

5) 알림 타입(최소 4종)

알림 타입을 구분할 수 있게 라벨/아이콘으로 표현:

마감 임박 : D-7/D-3/D-1 (가장 우선순위 높음)

모집 시작/재개

추가 확인 필요(서류/자격)

조건 업데이트 권장(프로필 보완)

6) 상태별 화면
openLoading

아이템 3개 정도 스켈레톤(제목/서브텍스트 막대) 표시

openEmpty

중앙 정렬 안내문:

“새 알림이 없어요”

서브: “조건을 채우면 더 정확한 알림을 받을 수 있어요”

버튼: “조건 보완하기”(작은 primary)

7) 반응형 규칙

데스크탑: 팝오버는 알림 아이콘 아래로 우측 정렬해서 떨어지게

모바일: 팝오버 폭은 화면의 92% 정도로, 상단 오른쪽에서 내려오는 느낌

8) 마무리 산출물

Header / NotificationBell 컴포넌트 세트 완성

NotificationPopover 내부 구성 요소(아이템, 빈 상태, 로딩 상태)도 재사용 가능한 하위 컴포넌트로 분리

기존 사이트의 타이포/색상/라운드/그림자 시스템을 그대로 재사용 (새 색상 정의 금지)

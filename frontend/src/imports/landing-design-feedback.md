근데 너는 “디자인 관해서 조언”이 목적이니까, 내가 네 코드(HomePage 등) 기준으로 랜딩 디자인 피드백을 딱 찝어서 줄게. (현재 구조는 랜딩 퀄리티 꽤 좋게 잡혀있음)

홈(랜딩) 디자인 피드백 — 지금 상태에서 더 ‘앱 같아지는’ 개선
1) Hero는 좋다 → “신뢰/근거” 한 줄만 더 넣으면 완성

지금 Hero는 감성+CTA가 강함. 여기서 부족한 건 “왜 믿어야 하는가”임.

추천 추가: 근거 1줄 + 숫자/출처

예: “서울시 청년정책 ○○개를 한곳에” / “매일 업데이트” / “마감 임박 알림”

CTA 아래에 작은 텍스트로:

“가입 없이 둘러보기 가능” 또는 “30초면 내 정책 찾기”

👉 이 한 줄이 전환율 진짜 올려줌.

2) 스크롤형 Feature 섹션이 3개 이상이면, 중간에 “요약 카드”가 필요해

너는 Feature 1 검색, Feature 2 자격확인 … 이런 식으로 스토리텔링을 하고 있잖아.
중간에 사용자 입장에서 “그래서 뭐가 좋은데?”를 다시 리마인드 해주는 요약 블록 넣어줘.

“뱅고 3줄 요약” 카드 (아이콘 3개 + 한 줄 설명)

빠른 검색

맞춤 추천

마감 알림/저장

3) “정책 서비스”는 결국 카드가 핵심 → 카드 디자인 규격을 미리 고정해

지금 목업 카드들은 스켈레톤 느낌인데, 실제 정책 카드에 들어갈 핵심 요소를 정해두면 훨씬 프로덕션 같아져.

정책 카드 권장 규격:

상태 뱃지: 모집중/마감임박/상시

제목 1줄(최대 2줄)

한 줄 요약

핵심 조건 2~3개 칩(나이/지역/소득 같은)

마감일(없으면 “상시”)

액션: 저장(북마크) + 자세히

👉 이걸 컴포넌트로 확정해두면 서비스 전체 톤이 정리됨.

4) “필터”는 강조 잘했는데, 실제 UX에선 ‘선택된 필터 요약바’가 필요해

필터를 쓰는 서비스는 공통적으로:

상단에 선택된 필터 칩 줄 + 전체 초기화가 있어야 사용자가 안 헤맴.

“필터 적용 결과 N개”가 즉시 보여야 함.

즉, 정책 리스트 상단을:

선택 필터 칩들 + 초기화 + 정렬(마감순/추천순)
이렇게 잡으면 완성도 확 올라가.

5) 페이지가 ‘예뻐보이긴 하는데’ 앱 서비스 느낌을 주려면 하단 2개가 필요

랜딩에서 자주 빠지는 것들:

FAQ(아코디언): “정보는 어디서 가져와요?”, “업데이트 주기?”, “알림은 어떻게?”

Footer: 문의/약관/개인정보/출처(서울 열린데이터 등)

이거 넣으면 갑자기 “진짜 서비스”처럼 보임.

6) 애니메이션 자체는 좋음 → 다만 “너무 많이”면 산만해질 수 있어

너는 blob + gradient + bounce + motion reveal이 동시에 들어가 있어.
추천은:

Hero에만 blob+gradient 유지

아래 섹션은 motion reveal만 살리고, 반복 애니메이션은 줄이기
(랜딩이 고급스러워짐)
